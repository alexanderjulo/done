done - about

What is done?

At it's core done is just an API to manage and synchronise todos. You can imagine it a little like simplenote, only that simplenote is for notes and done is for todos.

This API is surrounded by a set of apps. The web app is responsive and works on smartphones, tablets and desktop, as long as they have a recent browser available. But as the API is accessible and documented, anyone can write an app for done. That is actually what I am hoping for. I am neither good with Java nor with ObjC, so I hope that people will write apps for Android and iOS at a certain point. Or even Windows, Linux or Mac apps. Why would you do that? Well, you could sell the app and make money of it. But maybe you just want the perfect client. Well, that is no problem either. You are free to do as you like.

Features

  • Unlimited Todos, Projects and Areas!
  • Obviously due dates
  • Notes and recurring tasks (unlimited)
  • Reminders
  • Responsive/mobile ready! Web app included
  • No API limits! (if you don't use the web app)
  • The hosted version is completely SSL-encrypted.

Design

done is following the GTD model. You can create tasks and either attach them to projects or areas. Areas are for long-term responsibilities like private. Projects can either stand on their own or can be attached to an area, i.e. design a todo app as project in your area work.

Tasks and projects can have due dates (but don't have to), areas can't.

Roadmap

Well, let's put it this way. There is a roadmap, but there is no time attached to it, yet. The plan is as follows:

  1. Add multi-user support
  2. Make the app mobile ready
  3. Finish the API-documentation
  4. Opensource the code
  5. Make the hosted service liable to pay costs

License & Pricing

For now the source code is private, as done is still in an early alpha state. Not all features are working as they are supposed to or even available. As done grows it will be open sourced.

This way anyone will be able to host an instance themselves. But I will also be hosting an instance here. This way people, who don't have a server or don't want to bother about installing done and keeping it running, can use the hosted version. I won't host it for free though. I will charge a one-digit dollar amount per month, to pay for hosting and my development time. This model is inspired by sentry.

This also means, as long as it is not opensourced, hosted done will be free! So sign up and try it out!
